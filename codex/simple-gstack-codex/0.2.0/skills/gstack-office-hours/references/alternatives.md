# Alternatives Generation

Always produce at least two distinct approaches before writing the design memo.

## Required Shape

```text
Approach A: Name
  Summary: 1-2 sentences.
  Effort: S | M | L | XL
  Risk: Low | Medium | High
  Pros:
  - ...
  Cons:
  - ...
  Reuses: existing code, patterns, or assets.

Approach B: Name
  ...
```

## Required Alternatives

- Minimal viable approach: fewest files, smallest scope, fastest useful artifact.
- Ideal architecture approach: best long-term shape if the idea proves worth building.
- Creative or lateral approach: optional for simple ideas, preferred for non-trivial ideas.

## Recommendation

Recommend one approach and tie the reason to the selected mode:

- Startup Mode: evidence, wedge sharpness, speed to demand validation, distribution path.
- Builder Mode: delight, shareability, learning value, fastest path to a demo.

Stop for user choice in interactive sessions before finalizing the design memo.
